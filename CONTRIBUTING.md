# Contributing

Welcome, and thank you for your interest in OpenToonz!

This document describes how you can contribute to the project. There are many ways you can contribute, it's not all about code!

## Asking Questions
You can ask questions by opening a [new issue here](https://github.com/opentoonz/opentoonz/issues/new).

Please keep the questions related to OpenToonz. If you are searching for a **how-to** use OpenToonz, we have an extensive manual you can read [here](https://opentoonz.readthedocs.io/).

We also have a [user forum](https://groups.google.com/forum/#!forum/opentoonz_en/categories) where you can ask and answer general questions. Before asking a question here, consider if your question may be better suited for the forum!

## Reporting Bugs
If you've found a problem, please send us a report so we can fix it! You can open a new report [here](https://github.com/opentoonz/opentoonz/issues/new).

## Feature Requests
Do you have an idea how we can improve OpenToonz? We'd love to hear it! You can post a feature request by opening a new report [here](https://github.com/opentoonz/opentoonz/issues/new).

### Search For an Existing Issue
Before you file a new bug report, feature request or question, please use the [search function](https://github.com/opentoonz/opentoonz/issues) to check that nobody has already posted it to prevent making duplicates or your issue may be closed.

## Pull Requests

* Fork or clone the OpenToonz repository to your account.
* To clone, open a command line and use:
- * `git clone git@github.com:your-github-account/opentoonz.git`
* * `git remote add upstream https://github.com/opentoonz/opentoonz.git`
- You should work with [branches](https://help.github.com/en/github/collaborating-with-issues-and-pull-requests/about-branches).
- - `git checkout -b your-branch-name`.
- - - The `your-branch-name` is the name of your changes.
- You can commit your changes using `git commit`, please use descriptive commit messages.
### Syncing Your Fork
- Use `git pull upstream master` or `git pull --rebase upstream master`.

### Committing Your Changes
- Use `git push origin your-branch-name`.
